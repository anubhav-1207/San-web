What is San?

San is a dynamically-typed, expression-oriented programming language with a hand-written recursive-descent parser and tree-walking evaluator. It is designed to be readable, minimal, and easy to embed or extend — without sacrificing real programming power.

San was built from scratch in Python as an exploration of language design and interpreter construction — from lexing raw source text all the way to evaluating an abstract syntax tree at runtime.

💡 San is the product of Project-Arc — a personal journey to understand how programming languages actually work, built on an Android tablet using Pydroid, one commit at a time.

fn fib(n) {
  if n <= 1 { return n }
  return fib(n - 1) + fib(n - 2)
}

print(fib(10))        // 55

Features

🔤

Handwritten Lexer

A clean tokenizer that handles keywords, literals, operators, and identifiers with precise error reporting.

🌲

Recursive Descent Parser

Fully hand-written parser producing a typed AST — no parser generators, no magic.

⚡

Tree-Walking Evaluator

Direct AST interpretation with proper scoping, closures, and environment chaining.

🔁

First-Class Functions

Functions are values. Pass them around, return them, and close over their environment.

🧱

Lexical Scoping

Block-level scoping with nested environments — variable resolution follows the classic chain model.

🛡️

Runtime Error Handling

Descriptive errors with line information so you know exactly where things went wrong.

Functions & Closures

// Defining a function
fn add(a, b) {
  return a + b
}

// Closures
fn counter() {
  let count = 0
  fn increment() {
    count = count + 1
    return count
  }
  return increment
}

let c = counter()
print(c())  // 1
print(c())  // 2

Architecture

San is structured as a classic pipeline: each stage transforms the source into something the next stage can work with.

Source Text
    │
    ▼
┌─────────┐
│  Lexer  │  Tokenises raw characters into a flat stream of tokens
└────┬────┘
     │  Token Stream
     ▼
┌──────────┐
│  Parser  │  Recursive-descent parser builds a typed AST
└────┬─────┘
     │  Abstract Syntax Tree
     ▼
┌─────────────┐
│  Evaluator  │  Tree-walker interprets the AST with an Environment chain
└─────────────┘
     │
     ▼
  Output / Side Effects
